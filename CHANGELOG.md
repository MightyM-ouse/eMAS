### Changed

- all AI recommendations in the reviewed decision register are adopted as approved design decisions;
- authority policy now governs approved amendments pending consolidated document revisions;
- repository README and documentation index now route through canonical governance records;
- contribution and pull-request workflows now require DecisionIds, authority checks, change-class approvals and delivery-state accuracy;
- decision-register documentation now reflects the approved outcome rather than an open-question state;
- LLM context now distinguishes approved decision, synchronized documentation, implementation, verification and release;
- phase terminology is standardized as Pre-Sales Assessment, Pre-Migration Readiness and Post-Migration Verification;
- approved result wording now uses `Accepted Exceptions` consistently;
- LLM evidence provenance now uses Observed, CustomerProvided, Imported, Derived and Assumed;
- evaluation status and RAG are explicitly separate;
- classification terminology separates technical standard, regional implementation, procedure context and source presentation;
- ASMF is governed as procedure context rather than a technical submission format;
- the AI-assistant overview is labelled as generated and non-authoritative.
