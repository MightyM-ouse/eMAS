## 3. Global Design Rules

1. Column headers are fixed and protected.
2. Every finite value set uses a dropdown from `Value_Lists`.
3. Free text is permitted only where inherently necessary.
4. No comma-separated multi-value relationships are permitted.
5. Repeated relationships use link tables.
6. Editable `IsActive` is not used for lifecycle control.
7. Rule runtime eligibility is calculated.
8. `Rule_Index` is generated by VBA.
9. Only Effective rules enter controlled runtime JSON.
10. PowerShell does not read the workbook.

## 43. Completion Criteria

The content model is ready for workbook implementation when:

- all tables and keys are approved;
- all code lists are approved;
- all dropdown-controlled fields are identified;
- no uncontrolled multi-value relationships remain;
- lifecycle and supersession are deterministic;
- rule-phase behavior is explicit;
- conditions normalize correctly;
- findings and recommendations are separated;
- exception policy behavior is approved;
- JSON Schema 1.0.0 represents all relationships;
- proof-of-concept rules pass validation and export.
