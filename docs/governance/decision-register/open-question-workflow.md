# Approved-Decision Implementation Workflow

All decision-register items are approved. This workflow controls their implementation and closure.

## 1. Select work

Choose approved items in dependency order using the decision log and traceability matrix.

## 2. Prepare change

Identify Decision IDs, requirements, canonical artifacts, implementation, tests, compatibility and required reviewers.

## 3. Implement

Update the highest-authority affected artifact first, then architecture/schema, context/skills, implementation, tests and operations guidance.

## 4. Verify

An item is implementation-complete only when:

- canonical documents are synchronized;
- implementation exists where required;
- tests pass;
- traceability is updated;
- obsolete artifacts are marked superseded;
- required SME or owner evidence is recorded.

## 5. Release

Only verified items enter a controlled release. Approval alone does not establish implementation or release completion.
