# eMAS Change-Authority Matrix

**Version:** 1.0  
**Status:** Effective  
**Decision reference:** DEC-2026-006

| Change type | Owner | Required approvers |
|---|---|---|
| Enterprise scope or product boundary | Product Owner | Product Owner and Solution Architect |
| Regulatory content or classification rule | Regulatory SME | Regulatory SME and Product Owner |
| Effort or confidence weights and thresholds | Migration SME | Migration SME and Product Owner |
| Runtime JSON Schema | Technical Architect | Product Owner and PowerShell Lead |
| PowerShell engine or phase orchestration | PowerShell Lead | Technical reviewer and affected area owner |
| XLSM logical model or VBA | Technical Architect | Product Owner; Corporate IT for signing controls |
| Report template or controlled wording | Product Owner | QA/Validation reviewer and phase owner |
| WPF interface | Technical Architect | PowerShell Lead and affected phase owner |
| Test strategy or release evidence | QA Lead | Area owner and Product Owner |
| Canonical documentation | Documentation Owner | Applicable area owner |
| LLM context or operational skill | Documentation Owner | Applicable canonical-source owner |

## Control rules

- Canonical changes must reference a Decision ID or approved requirement.
- Regulatory content generated with LLM assistance remains Draft until normal SME approval.
- No approver may approve a material change solely because an AI recommendation exists.
